# 다중 사용자 환경을 위한 템플릿 동기화 계획

여러 명의 직원이 사용하는 웹사이트라면, 템플릿 설정(PDF 및 좌표)이 모든 사용자에게 원활하게 동기화되어야 합니다.

현재 템플릿 데이터(PDF 포함)가 약 83MB로 매우 커서, 파이어베이스 데이터베이스(Firestore, 1MB 제한)에는 실시간으로 저장할 수 없습니다. 파이어베이스 스토리지를 도입하면 전체 앱 구조를 바꿔야 하고 트래픽 비용이 발생할 수 있습니다.

따라서 **현재 사용 중이신 깃허브 페이지(무료 호스팅)를 100% 활용하는 가장 효율적인 대안**을 제안합니다.

## 제안하는 구현 방식 (관리자 배포 방식)

1. **관리자용 [웹에 배포하기] 버튼 추가**
   - 템플릿 관리 창에 **[웹에 배포하기]** 버튼을 만듭니다.
   - 관리자가 이 버튼을 누르면 현재까지 작업한 모든 템플릿과 좌표 정보가 담긴 `templates_data.js` 파일이 다운로드됩니다.
   - 관리자는 다운로드된 파일로 기존 파일을 덮어씌운 뒤, **깃허브 데스크탑으로 Push** 합니다. (이것으로 웹사이트 전체가 업데이트됩니다!)

2. **일반 사용자용 자동 업데이트 로직 적용**
   - 현재 시스템은 일반 사용자가 사이트에 접속할 때, 자신의 컴퓨터에 이미 템플릿이 저장되어 있으면 웹사이트가 업데이트되어도 예전 템플릿을 계속 사용하는 문제가 있습니다.
   - 이 문제를 해결하기 위해, 사이트 접속 시 **서버(templates_data.js)에 있는 템플릿 정보가 내 컴퓨터에 있는 정보와 다르면 강제로 최신 버전을 덮어씌우도록(업데이트)** 코드를 수정합니다.
   - 이렇게 하면 일반 직원들은 그저 사이트를 **새로고침**하기만 해도 관리자가 업데이트한 최신 양식과 좌표를 바로 사용할 수 있습니다.

## 요약
- **관리자**: 양식 작업 -> [웹에 배포하기] 클릭 -> 깃허브 데스크탑 Push
- **일반 직원**: 웹사이트 새로고침 -> 템플릿 자동 업데이트 완료!

파이어베이스 추가 요금이나 복잡한 설정 없이, 현재 환경에서 가장 완벽하게 다중 사용자를 지원하는 방법입니다. 
승인해 주시면 바로 코드를 수정하겠습니다!
